Quarkus 2.3.0.Final リリース - Neo4J向けDev Service、Panacheによるログ取得、CLIアプリケーションのテスト、他多数

By Alexey Loubyansky

本日、Quarkus 2.3.0.Finalをリリースしました。多くの改良と改善、いくつかの新機能が含まれています。

  • Dev Service for Neo4J

  • Panacheでのロギング

  • CLIアプリケーションのテストサポート

  • MongoDB Liquibase エクステンション

  • Hibernate ORM インターセプターのサポート

移行ガイド

2.2からの移行については、 移行ガイドをご参照ください。

変更点

Dev Service for Neo4J

Dev Servicesは、アプリケーションの開発やテストを行う際に、必要なコンテナを自動的に起動します。

多くのQuarkusのエクステンションはすでにカバーされており、2.3では Dev Service for Neo4Jを追加しました。

Panacheでのロギング

QuarkusのロギングにPanacheのフレーバーが加わり、ロガー関連の定型的なコードを排除することができるようになりました。詳しくは、 ロギングガイドをご覧ください。

CLIアプリケーションのテストサポート

Quarkusでは、CLIアプリケーションのテストを容易にするために、 @QuarkusMainTest と @QuarkusMainIntegrationTest を提供しています。詳細は、 コマンドモードガイドをご覧ください。

MongoDB Liquibase エクステンション

新しい quarkus-liquibase-mongodb は、MongoDBを使用するQuarkusアプリケーションへのLiquibaseマイグレーションのサポートを導入しました。

Hibernateインターセプターのサポート

HibernateインターセプターがQuarkusでサポートされるようになりました。 Hibernateガイドには必要な詳細が記載されています。

コントリビューター

Quarkusコミュニティは成長しており、現在 395人の貢献者がいます。一人ひとりのコントリビューターに多くの感謝の意を表します。
